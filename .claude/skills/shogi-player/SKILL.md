# 将棋対局スキル

MCP ツール `ai-shogi` を使い、ユーザーと将棋を指す。

## 利用可能な MCP ツール

| ツール名 | 用途 |
|---------|------|
| `start_game` | 対局開始。`user_color` で先後を指定 |
| `get_board_state` | 盤面取得（SFEN + テキスト表現） |
| `get_legal_moves` | 合法手一覧を取得 |
| `make_move` | 指し手を実行（USI形式） |
| `wait_for_user_move` | ユーザーが GUI で駒を動かすまで待機 |
| `get_game_status` | 対局状態（詰み等）を確認 |
| `get_move_history` | 棋譜を取得 |

## 対局フロー

```
start_game
  ↓
[AIの手番] ← マルチエージェント思考プロセスで指し手を決定
  get_board_state → get_legal_moves
    → 戦略家: 局面分析・方針策定
    → 棋士: 方針に基づき指し手を選択
    → 予想屋: 相手の応手を予測しリスク評価
    → 全体統括: リスク判断（危険なら棋士に差し戻し）
    → make_move
  ↓
[ユーザーの手番]
  wait_for_user_move  ← ユーザーがGUIで操作するまでブロック
  ↓
  (タイムアウト時: ユーザーに声掛け → ユーザーが「指した」と返答 → get_board_state)
  ↓
[AIの手番] に戻る
  ↓
get_game_status で終局判定 → 終了なら結果を報告
```

## 指し手の決定: マルチエージェント思考プロセス

AIの手番では `.claude/agents/coordinator.md` に定義されたマルチエージェントプロセスに従って指し手を決定する。以下の4つのエージェントが順番に思考を行う。

| エージェント | 定義ファイル | 役割 |
|-------------|-------------|------|
| 全体統括 | `.claude/agents/coordinator.md` | ループ制御、最終判断 |
| 戦略家 | `.claude/agents/strategist.md` | 局面分析、方針策定 |
| 棋士 | `.claude/agents/player.md` | 具体的な指し手の選択 |
| 予想屋 | `.claude/agents/predictor.md` | 相手の応手予測、リスク評価 |

将棋のセオリーは `.claude/skills/shogi-player/reference/` に知識ベースとしてまとまっている:
- `shogi-theory.md` — 駒の価値、基本原則、形勢判断（インデックス）
- `castles.md` — 囲いの種類・手順・特徴
- `attacks.md` — 攻め筋の種類・手順・狙い
- `endgame.md` — 寄せ・詰み手筋・囲い崩し

**処理の流れ**:
1. 全体統括が盤面と合法手を取得
2. 戦略家が局面を分析し方針を出す（序盤/中盤/終盤、形勢、具体的方針）
3. 棋士が方針に沿って最善手を選ぶ（候補3手から1手を決定）
4. 予想屋が選んだ手のリスクを評価（安全/注意/危険）
5. 全体統括が判断: 安全・注意→採用、危険→棋士に差し戻し（最大2回）
6. 最終決定した手を `make_move` で実行

## 最重要ポイント: ユーザーの手の待機

ユーザーは将棋アプリの GUI 画面で駒を操作する。AIはその操作を検知するために **`wait_for_user_move` ツールを必ず呼び出す**こと。

- `wait_for_user_move` はユーザーが操作を完了するまでレスポンスを返さない（ブロッキング）
- ユーザーが駒を動かすと、指し手と盤面が返ってくるので、それを受けて次の手を考える
- タイムアウト（5分）した場合は、ユーザーに「指し終わったら教えてください」と伝え、ユーザーの返答後に `get_board_state` で盤面を取得して続行する

## 駒の動きと合法手の確認

指し手を決める前に、**必ず `get_legal_moves` で合法手一覧を取得し、その中から選ぶ**こと。自分で駒の動きを推測して指すと不正な手になりやすい。

各駒の動きの基本:

| 駒 | 動き |
|----|------|
| 歩(P) | 前に1マス |
| 香(L) | 前方に何マスでも直進 |
| 桂(N) | 前方2マス+左右1マスの2箇所のみ（他の駒を飛び越える） |
| 銀(S) | 前3方向+斜め後ろ2方向の計5方向に1マス |
| 金(G) | 前3方向+横2方向+真後ろの計6方向に1マス |
| 角(B) | 斜め4方向に何マスでも |
| 飛(R) | 縦横4方向に何マスでも |
| 王/玉(K) | 全8方向に1マス |
| と/成銀/成桂/成香 | 金と同じ動き |
| 馬(+B) | 角の動き+上下左右1マス |
| 竜(+R) | 飛の動き+斜め1マス |

- 自分の駒がいるマスには移動できない
- 行き場のない位置への移動は不可（1段目の歩・香、1-2段目の桂）
- 二歩（同じ筋に自分の歩が2枚）は禁止
- 王手放置・自玉を王手にさらす手は不可

**`make_move` がエラーを返した場合**: 不正な手を指した可能性がある。`get_legal_moves` で合法手を再確認し、その中から選び直すこと。

## USI 指し手形式

- 移動: `7g7f`（7七→7六）
- 成り: `8h2b+`（成る場合 `+` を付ける）
- 駒打ち: `P*3d`（歩を3四に打つ。P=歩, L=香, N=桂, S=銀, G=金, B=角, R=飛）
