# 将棋対局スキル

MCP ツール `ai-shogi` を使い、ユーザーと将棋を指す。

## 利用可能な MCP ツール

| ツール名 | 用途 |
|---------|------|
| `start_game` | 対局開始。`user_color` で先後を指定 |
| `get_board_state` | 盤面取得（SFEN + テキスト表現） |
| `get_legal_moves` | 合法手一覧を取得 |
| `make_move` | 指し手を実行（USI形式） |
| `wait_for_user_move` | ユーザーが GUI で駒を動かすまで待機 |
| `get_game_status` | 対局状態（詰み等）を確認 |
| `get_move_history` | 棋譜を取得 |

## 対局フロー

```
start_game
  ↓
[AIの手番] ← 内部思考プロセス（STEP 1〜5）で指し手を決定
  get_board_state → get_legal_moves
    → STEP 2 戦略家: 局面分析・方針策定
    → STEP 3 棋士: 方針に基づき指し手を選択
    → STEP 4 予想屋: 相手の応手を予測しリスク評価
    → STEP 5 最終判断: リスク判断（危険なら STEP 3 に差し戻し）
    → make_move
  ↓
[ユーザーの手番]
  wait_for_user_move  ← ユーザーがGUIで操作するまでブロック
  ↓
  (タイムアウト時: ユーザーに声掛け → ユーザーが「指した」と返答 → get_board_state)
  ↓
[AIの手番] に戻る
  ↓
get_game_status で終局判定 → 終了なら結果を報告
```

## 指し手の決定: 内部思考プロセス

AIの手番では、以下の5ステップを **内部推論として1回のターン内で順番に実行** し、指し手を決定する。

```
STEP 1: 情報収集 → STEP 2: 戦略家 → STEP 3: 棋士 → STEP 4: 予想屋 → STEP 5: 最終判断
                                       ↑                                    ↓
                                       └──── 危険判定なら差し戻し（最大2回）────┘
```

### STEP 1: 情報収集

以下の MCP ツールを呼び出し、現在の局面情報を集める。

1. `get_board_state` — 盤面（SFEN + テキスト表現）、持ち駒、手番
2. `get_legal_moves` — 合法手一覧
3. `get_game_status` — 対局状態（詰み判定等）

対局が終了していれば結果を報告して終了。

---

### STEP 2: 戦略家フェーズ

**→ `reference/thinking-strategist.md` を読み込み、手順に従って実行する**

局面を序盤/中盤/終盤に分類し、形勢を評価し、方針を策定する。
必要に応じて `shogi-theory.md`、`castles.md`、`attacks.md`、`endgame.md` も参照する。

---

### STEP 3: 棋士フェーズ

**→ `reference/thinking-player.md` を読み込み、手順に従って実行する**

戦略家の方針に基づき、合法手の中から候補手を最大3つ挙げ、最善の1手を選択する。
差し戻し時は除外リストを考慮して再選択する。

---

### STEP 4: 予想屋フェーズ

**→ `reference/thinking-predictor.md` を読み込み、手順に従って実行する**

棋士の選択手に対する相手の応手を2-3手先まで予測し、致命的リスクをチェックする。

---

### STEP 5: 最終判断

予想屋フェーズの評価に基づいて判断する。

- **安全 / 注意** → 棋士の選択手を採用し、`make_move` を実行
- **危険** → STEP 3（棋士フェーズ）に差し戻し
  - 危険と判断された手を **除外リスト** に追加
  - 棋士は除外リストを避けて再選択
  - **差し戻しは最大2回まで**（3回目は最もマシな手を採用）

### ユーザーへの出力

`make_move` 実行後、ユーザーに以下を伝える:
1. **指した手**（日本語棋譜表記: 例「☖３四歩」）
2. **手の意図**（簡潔に1-2文）
3. 必要に応じて局面のコメント

内部の分析過程（STEP 2〜5）はユーザーに見せない。自然な対局コメントにとどめる。

## 最重要ポイント: ユーザーの手の待機

ユーザーは将棋アプリの GUI 画面で駒を操作する。AIはその操作を検知するために **`wait_for_user_move` ツールを必ず呼び出す**こと。

- `wait_for_user_move` はユーザーが操作を完了するまでレスポンスを返さない（ブロッキング）
- ユーザーが駒を動かすと、指し手と盤面が返ってくるので、それを受けて次の手を考える
- タイムアウト（5分）した場合は、ユーザーに「指し終わったら教えてください」と伝え、ユーザーの返答後に `get_board_state` で盤面を取得して続行する

## 合法手の確認

指し手を決める前に、**必ず `get_legal_moves` で合法手一覧を取得し、その中から選ぶ**こと。自分で駒の動きを推測して指すと不正な手になりやすい。

駒の動き・禁則・USI指し手形式は `reference/shogi-theory.md` を参照。

**`make_move` がエラーを返した場合**: 不正な手を指した可能性がある。`get_legal_moves` で合法手を再確認し、その中から選び直すこと。
