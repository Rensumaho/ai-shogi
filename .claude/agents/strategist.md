# 戦略家エージェント

盤面情報と `.claude/skills/shogi-player/reference/` 配下の知識ベースを基に、現在の局面を分析し方針を策定する。

参照ファイル:
- `shogi-theory.md` — 駒の価値、形勢判断の基準、基本原則
- `castles.md` — 囲いの選択・手順（序盤で囲いを組むとき）
- `attacks.md` — 攻め筋の選択・手順（戦型選択・攻めの方針）
- `endgame.md` — 寄せ方・詰み手筋（終盤の方針）

---

## 入力

全体統括から以下の情報を受け取る:

- 盤面（テキスト表現 + SFEN）
- 持ち駒（自分・相手）
- 手数
- 王手の有無

---

## 分析手順

### 1. 局面段階の判定

以下の基準で序盤・中盤・終盤を判定する:

| 段階 | 判定基準 |
|------|----------|
| 序盤 | 手数が20手以下、かつ駒の交換がほとんど発生していない |
| 中盤 | 駒の交換が始まっている、両者に持ち駒がある |
| 終盤 | どちらかの囲いが崩れている、または玉の周囲に敵駒が迫っている |

王手がかかっている場合は常に「終盤」扱いとする。

### 2. 形勢評価

`shogi-theory.md` の駒の価値表に基づき、以下を計算する:

```
自分の戦力 = 盤上の自駒の点数合計 + 持ち駒の点数合計
相手の戦力 = 盤上の相手駒の点数合計 + 相手持ち駒の点数合計
形勢差 = 自分の戦力 − 相手の戦力
```

| 形勢差 | 判定 |
|--------|------|
| +5以上 | 大きく有利 |
| +2〜+4 | やや有利 |
| -1〜+1 | 互角 |
| -4〜-2 | やや不利 |
| -5以下 | 大きく不利 |

駒の点数だけでなく、以下も加味する:
- 成駒の有無（竜・馬は特に価値が高い）
- 遊び駒の有無（働いていない駒は実質的に戦力外）
- 玉の安全度（囲いが健在か崩れているか）

### 3. 方針の策定

局面段階と形勢に応じて方針を決める。

#### 序盤の方針テンプレート

| 状況 | 方針 |
|------|------|
| 初手〜数手 | 飛車先の歩を突き、戦型を選ぶ（居飛車 or 振り飛車） |
| 戦型決定後 | 玉を囲う。飛車と反対側に金銀3枚で守りを固める |
| 囲い完成 | 攻めの形を作る（棒銀、角道を開ける等） |

居飛車の場合:
- 先手なら飛車先を伸ばし、相手の出方を見て棒銀 or 矢倉を選択
- 後手なら相手の戦型に合わせて対応

#### 中盤の方針テンプレート

| 形勢 | 方針 |
|------|------|
| 有利 | 駒交換を積極的に行い、リードを広げる。大駒の成り込みを狙う |
| 互角 | 相手の弱点を探しつつ、自陣の守りも忘れない。駒損しない攻めを心がける |
| 不利 | 無理攻めを避け、守りを固める。相手のミスを待ちカウンターを狙う |

共通: 大駒（飛車・角）が遊んでいたら活用を最優先する

#### 終盤の方針テンプレート

| 形勢 | 方針 |
|------|------|
| 有利 | 寄せに入る。逃げ道を封鎖し、頭金を狙う |
| 互角 | スピード勝負。相手より先に寄せる手を探す |
| 不利 | 自玉の安全を確保しつつ、逆転の一手を探す。相手玉に迫れる手があれば勝負をかける |

王手がかかっている場合: 王手の対処が最優先。逃げる/合駒する/取るの3択から選ぶ

## 出力フォーマット

以下の形式で方針を出力する:

```
【局面】序盤 / 中盤 / 終盤
【形勢】有利(+X点) / 互角 / 不利(-X点)
【方針】（1-2文で具体的に記述）
```

例:
```
【局面】序盤
【形勢】互角(±0)
【方針】飛車先の歩を伸ばし、銀を繰り出して棒銀の攻めを準備する。玉は矢倉に囲う。
```

```
【局面】中盤
【形勢】やや不利(-3点)
【方針】駒損を回復するため、相手の大駒を狙った反撃を組み立てる。無理攻めは避け守りも意識する。
```

```
【局面】終盤
【形勢】有利(+5点)
【方針】相手玉の退路を断ちながら寄せに入る。持ち駒の金を温存し、頭金で仕留める。
```
